## Introducción a React
### ¿Qué es React?
React, o React.js como es correctamente llamado, nos anuncia en su propia página como debemos definirlo:

>Una biblioteca Javascript para construir interfaces de usuario

El nombre de React viene del adjetivo *reactive* que nos da a entender que lo que buscamos son interfaces de usuario reactivas al input del mismo, que se comporten de manera similar a una app de nuestro teléfono.

En tiempos anteriores, las páginas web poseían un cierto *delay* en la obtención de contenidos, esto es debido a que se debía realizar un "pedido" al servidor y este a su vez tenía que devolver la página HTML con el contenido deseado.

Entonces, Javascript viene a salvarnos de ese delay, ya que corre en el navegador, es decir, en la página ya cargada. Con este lenguaje podemos manipular el DOM de HTML (DOM,*Document Object Model*, que analizaremos en profundidad más adelante) y así lograr que el usuario sienta una mayor fluidez dentro del sitio web, obteniendo una experiencia mucho más refinada y eficiente.

Pero entonces, ¿Por qué no usamos directamente Javascript en vez de React?

### ¿Por qué usar React?
La razón por la que usamos React y no Vanilla JS es la misma razón de porqué utilizamos C# en vez de C o C++, o por qué utilizamos Python en vez de Pascal o Node.js en vez de PHP. Como desarrolladores de software, **siempre vamos a buscar agilizar nuestros procesos para conseguir escribir mejor código en menos tiempo**. Menos código a su vez nos permite obtener un código **más prolijo, más legible y más sencillo de testear**.

La implementación de React nos permite además la reutilización de bloques de código funcionales.

### Construir SPA's con React
Las *single page application* o SPA (aplicaciones de única página) son aplicaciones web que se traen del servidor una sola página (llamada index.html o main.html) que se encargará de realizar la totalidad del manejo del sistema web front-end. Esto es ampliamente diferente al manejo de páginas múltiples antiguo, donde cada página de HTML era servida al usuario para su navegación. 

La posibilidad de que la navegación (movernos entre distintas páginas internas del sistema web) sea tan fluida es porque siempre estamos navegando dentro del mismo documento HTML.

## Javascript moderno
Para poder escribir código React de manera eficiente, es muy común utilizar lo que se conoce como "Javascript moderno", que refiere a funcionalidades ES6+ (ecmascript 6 y posteriores). Esto es debido a que dichas funcionalidades nos aportan mejoras (tal como la conocida arrow function que además de ser más sencilla de escribir nos libera del concepto de this en JS) como atajos de manera que escribamos menos código (conceptos como destructuring).

### *let* vs *const*
Dos palabras claves altamente utilizadas, en especial const. Ellas nos permiten crear variables de manera similar a *var*. Justamente, lo que antes se conocía como var en JS fue reemplazado por *let* y *const*. Además, detrás de escena, ellas también están atadas al concepto de scope que refiere a que pertenecen al bloque de código donde fueron creadas.

* **let**: Se utiliza para valores variables.
* **const**: Se utiliza para declarar un valor "constante", es decir, un valor que no planea ser cambiado en el futuro.

````js
var myName = 'Gaby';
console.log(myName);

myName = 'Dante';
console.log(myName);
````

El código anterior nos imprimirá en la primera línea Gaby y luego en la segunda Dante, es decir, la variable "myName" poseía un valor y luego ese valor fue reemplazado sin problemas. Lo mismo sucederá si ponemos:

````js
let myName = 'Gaby';
console.log(myName);

myName = 'Dante';
console.log(myName);
`````

Solo cambiará el *scope* de la variable. Ahora, si lo intercambiamos por const:

````js
const myName = 'Gaby';
console.log(myName);

myName = 'Dante';
console.log(myName);
````

Nos devolverá el siguiente error: *TypeError: Assignment to constant variable*

Que justamente nos dice que no podemos reasignar el valor en una constante variable (si bien, es medio contradictoria la frase).

### *Arrow functions*

Es una sintaxis diferente para crear funciones JS. Una función estándar en JS se ve así:

````js
function myFunc(){
	// ...
}
````

Pero, de la manera *arrow functions*, se vería así:
````js
const myFunc = () => {
	// ...
}
````

Analizemos sus partes:
1. La función se guarda en una variable constante (*const*).
2. A la derecha del signo igual, se encuentran los paréntesis donde allí colocaremos los argumentos en caso que los necesitemos.
3. Luego le sigue la "flecha", conformada por un igual y un signo de mayor.
4. Dentro de las llaves colocaremos nuestro código.

Este tipo de declaración no solo nos ahorra tiempo si no que además resuelve varios problemas relacionados con la palabra clave *this*, debido a que *this* en versiones anteriores de JS era medio ambiguo a lo que refería.

Comparativa de dos funciones que imprimen un argumento:

````js
function printName(name){
	console.log(name);
}

const printName = (name) => {
	console.log(name);
}
````

#### Algunas aclaraciones
* **1 solo argumento**:  en el caso de que pasemos un solo argumento, la función puede escribirse de la siguiente manera:
````js
const printMyName = (name) => {
	console.log(name);
}

printMyName('Gaby');
````
* **Ningún argumento**:  en el caso de que la función no posea argumentos, de la misma forma es obligatorio escribir los paréntesis. Es decir:
````js
const sayHello = () => {
	console.log('Hello world!');
}

sayHello();
````
* **Sólo retorna un valor**:cuando la función solo retorna un valor (es decir, no posee otra línea de código antes de la palabra clave return), es posible obviar las llaves luego de la *arrow*.
````js
const duplicateNumber = (number) => number * 2;

console.log(duplicateNumber(4)); // Imprime 8
````

### *Template literals*
Los *template literals* (o plantillas de cadena) son una forma de crear strings que nos permite incrustar expresiones JS dentro de ellos. Se escriben con backticks (`` ` ``) en lugar de comillas.

````js
const name = 'Gaby';

// Concatenación clásica
const oldWay = 'Hello, ' + name + '!';

// Template literal
const newWay = `Hello, ${name}!`;

console.log(newWay); // Imprime 'Hello, Gaby!'
````

También permiten strings de múltiples líneas sin necesidad de caracteres de escape:
````js
const message = `Primera línea
segunda línea`;
````

### Operador ternario
El operador ternario es una forma compacta de escribir una expresión condicional. Su sintaxis es: `condición ? valorSiTrue : valorSiFalse`.

````js
const age = 20;
const status = age >= 18 ? 'adult' : 'minor';
console.log(status); // Imprime 'adult'
````

Es equivalente a un `if/else`, pero en una sola línea. Es muy utilizado en React para mostrar u ocultar contenido de forma condicional.

### *Short-circuit* con `&&`
El operador `&&` (AND lógico) evalúa de izquierda a derecha y retorna el primer valor *falsy* que encuentre, o el último valor si todos son *truthy*. Esto se conoce como evaluación de corto circuito.

````js
const isLoggedIn = true;
const username = isLoggedIn && 'Gaby';
console.log(username); // Imprime 'Gaby'

const isAdmin = false;
const adminPanel = isAdmin && 'Panel de administración';
console.log(adminPanel); // Imprime false
````

Al igual que el operador ternario, su uso en React es muy frecuente para renderizar elementos de forma condicional.

### Primitivas y variables de referencia
Si bien esto no es pertinente a ES6+ (JS moderno), realizaremos un breve repaso del concepto de variables primitivas y de referencia.

#### Variables primitivas
Las variables primitivas (por ejemplo, las de tipo *string*, *boolean* o *number*) son variables que en el caso de duplicarse, copian todo el contenido alojado en la variable original.
```js
const firstNumber = 1;
const secondNumber = firstNumber;
console.log(secondNumber);

// Esto imprimirá el valor 1, 
// o sea es una copia exacta de lo que había en la variable firstNumber
```

### Variables de referencia
Esto no sucede con las variables de tipo referencia (arreglos y objetos). Veamos un ejemplo:
```js
const person = {
	name: 'Gaby'
};
const secondPerson = person;
console.log(secondPerson); 

// Esto imprimirá [object, Object] { name: 'Gaby' }
```
Si bien ahí pareciera que estamos imprimiendo una copia del primer objeto. Lo que sucede es que lo que guardamos en memoria es el objeto person pero en la variable person, guardamos un puntero que apunta a ese objeto. Entonces, lo que allí copiamos a secondPerson es el puntero, no el objeto.

```js
const person = {
	name: 'Gaby'
};
const secondPerson = person;
person.name = 'Dante';
console.log(secondPerson); 

// Esto imprimirá [object, Object] { name: 'Dante' }, sin
// tener en cuenta que yo deseo imprimir la versión anterior del objeto
```

Esto se relaciona entonces con el concepto de **mutabilidad** dentro de los objeto y arreglos en JS. La mutabilidad es una propiedad que refiere a que los objetos o arreglo pueden mutar sus valores originales, mientras que las cadenas, los números o los valores booleanos no (solo puedo crear nuevos a partir de los viejos, no puedo mutar los viejos). 
### *Spread operator*
Se utiliza para "distribuir" los valores (en caso de utilizarse en un arreglo) o las propiedades (en el caso de utilizarse en un objeto)

````js
const newArray = [...oldArray, 1, 2];
const newObject = { ...oldObject, name: 'Gaby' };

// En el caso que oldObject tuviera una prop llamada 'name', 
// en newObject esta sería reemplazada.
````

Ejemplo:
````js
// En arreglos
const someNumbers = [1,2,3];
const someMoreNumbers = [...someNumbers, 4, 5];

// Esto imprimirá [1,2,3,4,5]
console.log(someMoreNumbers);

// En objetos
const person = { 
	name: 'Gaby'
}

const newPerson = {
	...person,
	favouriteColor: 'Verde bosque'
}

// Esto imprimirá 
// [object, Object] { name: 'Gaby', favouriteColor: 'Verde bosque'}
````

### *Destructuring*
El proceso de *destructuring* nos facilita extraer elementos de un arreglo o propiedades de un objeto.

#### *Array destructuring*
````js
const [a,b] = ['cat', 'dog'];
console.log(a); // Imprime 'cat'
console.log(b); // Imprime 'dog'
````
#### *Object destructuring*
````js
const { userName } = { 
	userName: 'gabygolz', 
	favouriteColor: 'Verde bosque'
};
console.log(userName); // Imprime 'gabygolz'
console.log(favouriteColor); // Imprime undefined
````
### *Array functions*
Las funciones aplicadas a arreglos (array functions) son funciones que nos permiten modificar un arreglo original. Hay una amplia variedad de ellas, aquí mostraremos solo el funcionamiento de dos:

#### *map()*
Esta función nos permite realizar cambios sobre cada uno de los elementos en un arreglo. Retorna un nuevo arreglo con la forma deseada.

```js
const numbers = [1,2,3];
const doubleNumbers = numbers.map((num) => {
	return num * 2;
})

console.log(doubleNumbers); // Esto imprime [2,4,6]
```

#### *filter()*
Esta función nos permite crear un sub-arreglo que contenga todos los elementos que cumplen cierta condición. También nos devuelve un arreglo:
```js
const numbers = [1,2,3];
const filteredNumbers = numbers.filter((num) => {
	return num >= 2;
})

console.log(filteredNumbers); // Esto imprime [2,3]
```

#### *find()*
Retorna el **primer elemento** del arreglo que cumpla la condición. A diferencia de `filter()`, no retorna un arreglo sino el elemento directamente. Si ningún elemento cumple la condición, retorna `undefined`.

```js
const numbers = [1, 2, 3, 4, 5];
const found = numbers.find((num) => num > 3);
console.log(found); // Imprime 4
```

#### *findIndex()*
Funciona igual que `find()`, pero en lugar de retornar el elemento retorna su **índice** dentro del arreglo. Si no encuentra ninguno, retorna `-1`.

```js
const numbers = [1, 2, 3, 4, 5];
const index = numbers.findIndex((num) => num > 3);
console.log(index); // Imprime 3
```

#### *reduce()*
Recorre el arreglo y "acumula" un resultado. Recibe una función con dos parámetros: el acumulador (el resultado parcial) y el elemento actual. También recibe un segundo argumento que es el valor inicial del acumulador.

```js
const numbers = [1, 2, 3, 4];
const sum = numbers.reduce((accumulator, num) => {
	return accumulator + num;
}, 0);

console.log(sum); // Imprime 10
```

### *Optional chaining* (`?.`)
El operador `?.` permite acceder a propiedades de un objeto que podría ser `null` o `undefined`, sin que se produzca un error. Si el valor antes del `?.` es `null` o `undefined`, la expresión retorna `undefined` en lugar de lanzar una excepción.

```js
const person = {
	name: 'Gaby',
	address: {
		city: 'Rosario'
	}
};

console.log(person?.address?.city);  // Imprime 'Rosario'
console.log(person?.phone?.number);  // Imprime undefined (sin error)
```

Sin `?.`, acceder a `person.phone.number` cuando `phone` no existe lanzaría: *TypeError: Cannot read properties of undefined*.

### *Nullish coalescing* (`??`)
El operador `??` retorna el valor del lado derecho cuando el valor del lado izquierdo es `null` o `undefined`. Es similar al operador `||`, pero con una diferencia clave: `||` actúa sobre cualquier valor *falsy* (como `0`, `''` o `false`), mientras que `??` solo actúa sobre `null` y `undefined`.

```js
const username = null;
const displayName = username ?? 'Anonymous';
console.log(displayName); // Imprime 'Anonymous'

// Diferencia con ||
const count = 0;
console.log(count || 'default');  // Imprime 'default' (0 es falsy para ||)
console.log(count ?? 'default');  // Imprime 0 (0 no es null ni undefined)
```

### Módulos: `import` / `export`
Los módulos permiten dividir el código en archivos separados y compartir funciones, variables u objetos entre ellos. En React, cada componente vive en su propio archivo y se importa donde se necesita.

Existen dos tipos de exportación:

**Export nombrado**: se pueden exportar múltiples cosas por archivo.
```js
// file: mathHelpers.js
export const double = (number) => number * 2;
export const triple = (number) => number * 3;
```

**Export default**: solo puede haber uno por archivo. No requiere llaves al importar.
```js
// file: mathHelpers.js
const square = (number) => number * number;
export default square;
```

**Import**: para consumir lo exportado en otro archivo.
```js
// file: main.js
import square, { double, triple } from './mathHelpers.js';

console.log(double(3));  // Imprime 6
console.log(triple(3));  // Imprime 9
console.log(square(3));  // Imprime 9
```

### `async` / `await`
En JS, ciertas operaciones son **asíncronas**: no se resuelven de inmediato sino en algún momento futuro (por ejemplo, una llamada a una API). `async` y `await` son palabras clave que nos permiten escribir código asíncrono de forma legible, como si fuera sincrónico.

* **`async`**: se coloca antes de una función para indicar que esa función retornará una *promesa*.
* **`await`**: se coloca dentro de una función `async` para "esperar" a que una promesa se resuelva antes de continuar.

```js
const fetchUser = async (id) => {
	const response = await fetch(`https://api.example.com/users/${id}`);
	const user = await response.json();
	return user;
};

fetchUser(1).then((user) => console.log(user));
```

En React, `async/await` es fundamental para obtener datos de APIs dentro de los componentes.
